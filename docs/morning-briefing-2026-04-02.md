## 二、最有价值的三件事

### 优先级 1️⃣：打通 AI 归因 → 让顾问有话可说

**现状：** 顾问跟客户讲策略，全靠个人经验积累的话术。

**如果 AI 能生成这样的内容：**
```
策略「低价转债」近一年收益 +12.28%，
超额收益 +8.48%（vs 理财基准）；
最大回撤 -8.05%，夏普比率 1.53。

核心驱动：年内转债市场两次触发「条款博弈」窗口，
管理人均在高位完成转股兑现，带来显著 alpha。
风险：若权益市场持续走弱，低价券绝对价格在105元以下
继续压缩，纯债底对净值支撑有限。

配置建议：持有期6个月以上，追求绝对收益的客户，
当前仍是较好介入时点。
```

**这是顾问最需要的武器。** 一键生成，不是占位符。

---

### 优先级 2️⃣：补全业绩归因 → 从展示工具升级为分析工具

当前详情页只有收益数字，没有"为什么涨/跌"。

**建议增加 Brinson 归因（至少是简化版）：**
- 超额收益 = 选股效应 + 行业配置效应 + 交互效应
- 用现有时序数据可以估算
- 不需要外部数据源

**加上风险仪表盘：**
- VaR (95%) / CVaR
- 历史最大回撤及恢复时间
- 压力测试场景：2022年熊市 / 2015年股灾 / 2020年3月疫情冲击

---

### 优先级 3️⃣：PDF 报告导出 → 顾问对客的实际工具

顾问跟客户谈完，需要给客户留一份"策略建议书"。

当前系统没有报告导出功能，这是顾问最实际的刚需。

**最小可落地方案：**
- 选定策略 → 生成格式化 PDF（策略名称/风险等级/历史业绩/适合人群/风险提示）
- 不需要复杂模板，用浏览器打印功能 + CSS print 样式即可快速实现

## 三、值得投入的中期功能

### 3.1 组合优化引擎（真实 Markowitz）

当前组合配置器是"权重滑块 + 手动调"，顾问不知道当前组合是否"够好"。

**如果加一个"智能优化"按钮：**
- 输入：选入的策略 + 各自权重上限
- 输出：有效前沿（Efficient Frontier）曲线 + 最优组合点
- 附带蒙特卡洛模拟（10000次随机组合，标注当前组合位置）

**业务价值：** 顾问可以向客户展示"我这个组合的风险收益比，在所有可能组合中处于什么位置"。

---

### 3.2 客户画像持久化

当前系统：顾问输入客户画像 → 推荐策略 → 关闭页面 → 数据消失。

**生产级需求：** 需要把每次推荐的客户画像 + 推荐结果存下来：
- 客户风险偏好记录
- 推荐历史（合规审计用）
- 后续跟踪（客户最终买了什么）

---

### 3.3 竞争分析模块

顾问经常被客户问："你们这个比别家怎么样？"

**如果系统里有：**
- 同类策略市场分布（我司 vs 竞品，在四象限图上的位置）
- 我司策略的差异化优势标注
- 简单的一句话定位

## 四、技术层面需要立即修的问题

### 问题 1：后端启动不稳定
- `npm run dev` 有时会静默退出
- 建议用 `pm2` 或 `npx tsx watch` 确保稳定运行

### 问题 2：前端端口漂移
- 5173 被占用后自动切 5174/5175，URL 不固定
- 建议固定前端端口或加一个启动提示页

### 问题 3：有些 API 是占位状态
- `/api/analyze` — 分析路由存在但不确定是否真实调用 AI
- `/api/explain` — 策略解读是否真实通需要验证
- 需要逐个接口过一遍"AI 真实调用 vs 占位返回"

## 六、更高视角的问题（值得思考）

这个系统最终要回答一个核心问题：

> **"光大资管的策略，适合什么样的客户，在什么市场环境下，应该怎么配？"**

当前系统更多在回答"策略是什么样的"，还没真正回答"这个客户的实际情况，应该怎么用这个策略"。

**这不是技术问题，是产品定位问题。**

如果把这个系统定位为"顾问的展示工具"——现在已经够用。
如果定位为"客户的决策支持系统"——还差得远。

**建议找时间跟业务侧（产品部/财富管理部）开一次会对齐：**
- 这个系统最终要给谁用？
- 他们的核心使用场景是什么？
- 成功标准怎么定义？

对齐了产品定位，迭代方向就清晰了。
