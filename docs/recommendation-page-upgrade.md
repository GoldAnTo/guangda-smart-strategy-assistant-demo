# 推荐结果页升级建议

## 页面目标

让推荐结果页从“给几个产品名”升级为“可解释、可比较、可沟通”的页面。

## 建议页面结构

1. 客户画像摘要
2. 决策摘要条
3. 优先方案
4. 备选方案
5. 暂不推荐方向
6. 推荐依据
7. 策略与基准对比
8. 风险提示
9. 转人工下一步动作

## 关键展示模块

### 决策摘要条
示例：
- 当前更适合稳健增厚型方案，优先考虑固收+，纯债作为低波备选。

### 推荐依据
建议固定显示：
- 客户画像依据
- 命中的规则依据
- 当前不推荐的原因

### 候选方案分层
- 优先方案：当前最匹配的主推方向
- 备选方案：可替代但匹配度略低的方向
- 暂不推荐：当前不建议优先考虑的方向

### 转人工模块
建议结构化返回：
- `shouldEscalate`
- `escalationType`
- `escalationReason`
- `nextAction`

## 风险提示

页面中必须单独留出明显区域展示：
- 产品净值可能随市场变化波动
- 历史表现不代表未来
- 正式推荐仍需结合客户适当性与人工确认
