## 一、项目背景

### 1.1 业务背景

光大资管（光大证券旗下资产管理业务）管理着多条策略产品线，涵盖纯债、固收+、现金增强等不同风险收益定位的产品。业务侧存在以下痛点：

- **产品数量多、维度杂**：策略之间从风险等级、期限、流动性到收益目标均有差异，客户或业务人员难以快速横向比较
- **推荐决策缺抓手**：给客户做策略配置建议时，缺乏可量化、可解释的工具支撑
- **AI能力未落地**：大语言模型在投研场景的潜力尚未被充分挖掘，缺乏实际应用案例

### 1.2 项目起源

本项目最初定位为"**光大资管智能策略推荐助手 Demo**"，旨在：

1. 搭建一个可对外/对内展示的策略产品可视化平台
2. 在此基础上逐步叠加 AI 驱动的智能推荐能力
3. 最终演进为可接入真实客户画像的智能投研辅助工具

项目由鲁慧凌发起，当前以 **20条策略产品的展示与对比** 为核心范围。

## 四、产品与数据

### 4.1 当前产品范围

| 数据源 | 产品数量 | 用途 |
|--------|----------|------|
| `strategy_show.zip` | 20条策略 | 当前主展示数据（Java API） |
| Node.js API | 382个产品 | 备选 / 完整产品目录（端口3003） |

> ⚠️ 当前展示的20条策略数据均为**演示数据**，不代表真实产品业绩。后续替换为真实数据时须合规处理。

### 4.2 20条策略分类（统计截至 `2026-03-30`）

策略大致分为以下方向：

- **纯债策略**：低波动、稳健收益
- **固收+策略**：股债混合、追求增厚收益
- **现金增强策略**：高流动性、灵活配置

每条策略的核心字段：
```
productId / productName / navCategory / riskLevel /
inceptionDate / statsPeriod / annualizedReturn /
maxDrawdown / volatility / sharpeRatio /
benchmark（基准对比数据）
```

### 4.3 评分模型结构（规划中）

建议总分 = 5 项子分之和：

| 维度 | 权重 | 说明 |
|------|------|------|
| 风险匹配分 | 30 | 客户风险偏好 vs 产品风险等级 |
| 期限匹配分 | 20 | 客户投资期限 vs 产品期限标签 |
| 流动性匹配分 | 20 | 客户流动性要求 vs 产品流动性特征 |
| 收益目标匹配分 | 20 | 客户目标（稳健/增厚/平衡）vs 产品定位 |
| 资产平衡分 | 10 | 已有资产配置情况 |

## 五、功能模块

### 5.1 策略目录页 `/`
- 20条策略列表，支持分类筛选（纯债/固收+/现金增强）
- 每条策略卡片展示核心指标（年化收益/最大回撤/夏普比率/风险等级）
- 点击进入策略详情页

### 5.2 策略详情页 `/products/:id`
- 完整策略指标展示
- 策略与基准对比图表
- 历史净值曲线（待接入）
- 风险提示与合规声明

### 5.3 方向对比页 `/compare`
- 最多5条策略同屏横向对比
- 固定对比维度：收益/回撤/波动/夏普/风险等级/期限/流动性
- 支持选中不同方向策略做对比分析

### 5.4 组合配置器 `/recommendation`
- 左侧策略库（分类过滤 + 搜索 + 点击选入）
- 右侧权重滑块分配，实时计算总权重
- 组合指标实时聚合（年化收益/波动率/最大回撤/夏普）
- 分散化效果评估
- AI 组合解读按钮（占位，待对接真实AI）

### 5.5 未来推荐引擎（规划中）
- 输入：客户画像（风险偏好/期限/流动性/收益目标/已有资产）
- 推理：评分模型 + 规则引擎 + LLM 综合决策
- 输出：分层推荐结果 + 可解释理由 + 合规提示
